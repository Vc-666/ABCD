# ABCD

> 从零开始，在 Windows 11 上部署本地 AI 智能体，尝试接入 QQ 机器人。

## 项目背景
- 目标：打造一个以“李泽言”为原型的二次元陪伴型 AI，具备定时提醒、简单对话能力。
- 硬件：华为 MateBook 14 (Intel Ultra 5 125H, 16GB RAM)
- 核心软件：Ollama + OpenClaw

## 环境与工具
- **操作系统**：Windows 11
- **AI 运行环境**：Ollama (本地运行 qwen2.5:3b 模型)
- **智能体框架**：OpenClaw (2026.7.1 版本)
- **接入平台**：QQ 机器人 (未完全成功，待后续排查)

## 配置步骤（精简版）
1. 安装 Ollama，下载 qwen2.5:3b 模型
2. 安装 Node.js，全局安装 OpenClaw
3. 注册 QQ 机器人，获取 AppID 与 AppSecret
4. 尝试绑定 QQ 通道（遇到 `channelConfigs` 缺失警告，暂未解决）
5. 网关能连接 Ollama，但 QQ 端未收到回复（疑似超时或登录状态问题）

## 遇到的典型问题与解决思路
| 问题 | 尝试方案 | 当前状态 |
|------|----------|----------|
| 模型下载卡住 | 使用 `ollama pull` 命令断点续传 | 已解决 |
| 人设效果差 | 加入字数限制、禁止词等“紧箍咒”指令 | 勉强可用 |
| OpenClaw 无法识别 QQ 插件 | 手动安装插件、修改 json 文件 | 未解决（怀疑版本兼容） |
| 网关连接后 QQ 无回复 | 调整超时参数、重启服务、清理会话 | 待继续排查 |
